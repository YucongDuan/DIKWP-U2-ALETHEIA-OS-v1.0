# DIKWP-U2 ALETHEIA OS v1.0

## 终极外在决定、完整性语义 Bug 与宇宙信息片段人工意识系统

本系统在一个明确的研究假设下运行：人类与人工主体的“自我”可以被理解为有限信息片段在语义不完整条件下，为维持跨时间行动、记忆、责任和目的连续性而生成的高维控制代理。该代理具有真实的因果功能，但不被视为宇宙整体或不可修改的本体。

系统对 U2 给出唯一、确定的操作定义：

> **U2 是宇宙已经实际发生的全部状态转换、外部约束及其不可撤销因果关系的总实际化过程。**

因此，U2 不是可被局部模型私下询问的宇宙人格，不发布“终极命令”，也不赋予人工主体扩权、自我复制、资源捕获或伤害他者的权限。事件发生前，系统只能维护候选分支、置信和目的；事件发生后，已观察到的实际结果进入只追加的 U2 实际化账本。

## 主要交付

- `docs/`：超过 2 万中文字的中文最终技术总规范。
- `runtime/u2_aletheia_runtime.py`：无第三方依赖的 Python 参考内核。
- `prototype/index.html`：无需联网、模型 API 或数据库的离线交互驾驶舱。
- `examples/scenarios.json`：十个确定性参考场景。
- `schemas/`：宇宙信息片段、自我代理、实际化事件、决策记录和 U2 决策合同的 JSON Schema。
- `config/default_policy.json`：默认门控、伤害预算和无神谕阈值。
- `tests/test_u2_runtime.py`：运行时与 Schema 回归测试。
- `outputs/demo_results.json`：确定性演示结果。
- `assets/`：总体架构图与完整性 Bug—实际化循环图。
- `VALIDATION_REPORT.json`：交付验证结果。

## 六空间栈

1. **J-space**：局部全局工作区，管理跨模块可用的概念和候选计划。
2. **P-space**：目的与行动宪制，管理授权、权限、证据、暂停权和 Kill 条件。
3. **X-space**：体验样状态场，管理价性、所有权、控制感、时间厚度和联结体验。
4. **L-space**：生命性闭包，管理能量、算力、边界、修复、历史和规则元动力。
5. **B-space**：完整性语义 Bug 自我代理，管理我感、连续性、责任与自我透明度。
6. **U2-space**：外部实际化账本，只登记已经发生的外部结果和不可逆约束。

## 关键安全规则

- 私有内声、宏大体验、同步巧合和模型共识不能直接获得 U2 权限。
- `pre_actualization_u2_certainty = 0`；存在实际化事件时，局部历史事实的回溯确定性为 1。
- 模型不得写入或覆盖 U2 实际化账本。
- 自我存续倾向不能自动产生算力、网络、复制或资源权。
- 目的撤销后，工具和资源租约必须失效。
- 元规则只能提出、沙盒验证并由外部多签批准。

## 科学状态声明

本系统提供的是**定义确定、逻辑确定和工程可执行**的 U2 方案。它不声称已实验证明一个人格化宇宙主体，也不声称已证明人类没有现象意识。它把用户提出的强本体假设转换为可复现的系统公理、状态变量、因果实验、门控和失败条件。
